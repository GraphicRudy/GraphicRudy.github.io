flagThe Challenge

A capable AI. Sharp edges everywhere around it.

Karl's NL-to-SQL core works. But v0.1 shipped with zero documented error states, no role gating, and vague mechanics users need to trust the system. For a product promising "no SQL required," hidden complexity broke the promise.

fork_rightKey Decisions

Four calls I'd defend in a design review.

Trade-offs, not polished outputs. These shaped v0.2.

admin_panel_settings
DEC / 01
Role-gate the Knowledge Guide into two tiers
expand_more
lightbulbReasoning
v0.1 had Critical and Auxiliary Instructions in one space. I split them: Critical stays admin-only; Auxiliary opens to Power Users so they author domain rules without an admin bottleneck.
balanceTrade-off
Two tiers is more complex than one. I accepted it because admin-only bottleneck on everything would kill Raj's workflow.
memory
DEC / 02
Define Auxiliary activation as tags + classifier + manual
expand_more
lightbulbReasoning
v0.1 said Karl "detects" domain without saying how. I proposed three layers: domain tags first, LLM classifier as fallback, manual session override as escape hatch.
balanceTrade-off
This is a product decision, not just a doc decision. If engineering has different intent, guide and prototype need rework. Flagged instead of silently committing.
delete
DEC / 03
Drop the fabricated "section numbering bug"
expand_more
lightbulbReasoning
v0.1 claimed Section 4 was used twice. It wasn't. Dropped rather than carrying a fabricated issue. The team's own audit had drifted from reality.
balanceTrade-off
My gap count no longer matches internal reference. Chose honesty over continuity. Future audits run against the live product, not a mental model.
label
DEC / 04
Don't call this "Agent Experience Design"
expand_more
lightbulbReasoning
Useful but AX-adjacent. Didn't define autonomy, confidence calibration, reasoning traces, or correction loops — the core of real AX. Called it Product UX for an AI product.
balanceTrade-off
AX is the hotter label and attracts more attention. Traded attention for credibility. A future Agent Behavior Spec would earn the label honestly.

handshakeMethod & Disclosure

Yes, I used AI. Here's the split.

This case study was produced in a working session with Claude. More useful to show the workflow than pretend four polished HTML files materialized from thin air.

account_circleWhat I owned

  • Scoping. The four deliverables, visual language, audience.
  • Every design decision. Personas, gap priority, IA, role tiers, activation mechanism.
  • Quality judgment. Which gaps held up. What to drop. What to defer.
  • Honesty calls. Dropping fabricated bugs. Refusing to call this AX.

smart_toyWhat Claude did

  • Wrote the HTML/CSS. All deliverables and this page, to my direction.
  • Drafted copy from specs. Audit findings, persona quotes, guide chapters.
  • Generated inline SVG mocks used in the user guide.
  • Pushed back on me. "Can we call this AX?" got a detailed no — which made the study better.

lock_openWhy disclose

  • Honest. Pretending solo unassisted wouldn't survive scrutiny.
  • The workflow is the skill. Directing AI at pace is a real craft.
  • Hiring has shifted. Most teams in 2026 assume AI-assisted output.
  • Filter. Reviewers put off by this aren't teams I want.
Working Heuristic

Remove Claude from this project and I'd still have the same diagnosis, priority matrix, and IA — it would just take two weeks instead of two hours. The decisions are mine. The speed and craft are the collaboration.
